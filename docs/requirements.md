## 1. 背景・目的

### 1.1 背景
利用者は最近麻雀を始めた初心者で、PCゲーム「雀魂（じゃんたま）」の段位戦でランクを上げることを目標としている。現状は **役を全く覚えておらず、点数計算もできない**。雀魂は点数計算を自動で行うため、段位を上げる上での最大のボトルネックは「**和了（あが）れないこと**」＝どんな役があるか分からず、手牌を見ても役に気づけないことである。

### 1.2 目的
初心者が **「役を覚える（暗記）→ 定着させる（クイズ）→ 手牌から役を見抜く（実戦力）」** という上達の流れを丸ごと練習できる、自分専用の学習アプリを提供する。視覚的に学べるよう、実際の牌画像を用いる。

### 1.3 成功の定義（プロジェクトゴール）
本アプリを継続利用することで、利用者が雀魂の段位戦で**初心→雀士→雀傑**と段位を上げられるだけの「役の知識」と「手牌から役を見抜く力」を身につけること。

## 2. 対象ユーザー

| 項目 | 内容 |
|------|------|
| 主要ユーザー | 本人（麻雀初心者）1名のみ |
| 利用形態 | 完全に個人利用・**非商用** |
| 利用デバイス | PC（主・雀魂と同じ端末）＋ スマートフォン（補助・スキマ学習） |
| 前提知識 | 牌の種類（萬子・筒子・索子・字牌）は概ね分かる。役・点数計算は未習得 |

---

## 3. 用語定義

| 用語 | 定義 |
|------|------|
| 役（やく） | 和了に必要な手牌の組み合わせパターン。本アプリの学習対象 |
| ティア | 役の難易度・優先度の区分。Tier1=初級、Tier2=中級、Tier3=上級 |
| 面子（メンツ） | 順子（連番3枚）・刻子（同種3枚）などの3枚1組 |
| 雀頭（ジャントウ） | 同種2枚の対子。和了形に1つ必要 |
| 状況役 | 手牌の構成ではなく、和了の状況・タイミングで成立する役（立直・一発・ツモ・海底/河底など）。**手牌の形だけからは判定できない** |
| 段位 | 雀魂のランク。初心 → 雀士 → 雀傑 → 雀豪 → 雀聖 → 魂天（各1〜3） |
| MVP | Minimum Viable Product。最初に作る最小限の動くバージョン |

## 4. スコープ

### 4.1 フェーズ全体像

| フェーズ | 内容 | 本書での扱い |
|---------|------|------------|
| **フェーズ1（MVP）** | 3クイズモード／保存なし／Vercelデプロイ | **詳細に定義（今回実装対象）** |
| フェーズ2 | 進捗保存 ＋ 苦手復習モード | 概要のみ |
| フェーズ3 | 段位別ロードマップ（静的＋個人化） | 概要のみ |
| フェーズ4（候補・未定） | 手牌認識クイズの完全ランダム化 | 概要のみ |

### 4.2 スコープ外（本プロジェクトで作らないもの）
- 点数計算・符計算機能（雀魂が自動で行うため不要）
- 対局機能・対戦AI・牌譜解析
- 鳴き（副露）形の表示（横向き牌など）。フェーズ1の手牌は門前（手の内）形で表現する
- マルチユーザー対応（フェーズ2でも単一ユーザーの簡易認証に留める）
- 多言語対応（日本語のみ）
- ネイティブアプリ化（Web のみ）

## 5. 機能要件

> 記法: `FR-x.y`。優先度 = ◎必須 / ○推奨 / △任意

### 5.1 共通

| ID | 要件 | 優先度 |
|----|------|-------|
| FR-0.1 | ホーム画面から3モード（暗記カード／4択クイズ／手牌認識クイズ）へ遷移できる | ◎ |
| FR-0.2 | ティア（初級／中級／上級）を選択して学習範囲を絞り込める。初期表示は初級 | ◎ |
| FR-0.3 | 牌は実際の牌画像（SVG）で表示する | ◎ |
| FR-0.4 | UIは全て日本語 | ◎ |
| FR-0.5 | フェーズ1では進捗・成績を**保存しない**（リロードでリセット） | ◎ |

### 5.2 暗記カードモード（FR-1）

| ID | 要件 | 優先度 |
|----|------|-------|
| FR-1.1 | 選択中ティアの役を1枚ずつカード表示する | ◎ |
| FR-1.2 | カード表面に「役名・読み・飜数」を表示する | ◎ |
| FR-1.3 | カードをめくる（タップ/クリック）と裏面に「成立条件の説明」と「例の手牌画像」を表示する | ◎ |
| FR-1.4 | 「次へ／前へ」でカードを移動できる | ◎ |
| FR-1.5 | カード順をシャッフルできる | ○ |

### 5.3 4択クイズモード（FR-2）

| ID | 要件 | 優先度 |
|----|------|-------|
| FR-2.1 | 1セッション＝10問。選択中ティアの役から出題する | ◎ |
| FR-2.2 | 出題形式は「成立条件 → 役名を4択」を基本とする。「役名 → 条件を4択」も混在可 | ◎ |
| FR-2.3 | 誤答選択肢は紛らわしい役（同ティア等）から選ぶ | ○ |
| FR-2.4 | 解答すると即時に正誤を表示し、正解の解説を表示する | ◎ |
| FR-2.5 | セッション終了時に正答数（例: 7/10）を表示する | ◎ |
| FR-2.6 | セッションを再挑戦できる | ◎ |

### 5.4 手牌認識クイズモード（FR-3）

| ID | 要件 | 優先度 |
|----|------|-------|
| FR-3.1 | 1セッション＝10問。手牌（13〜14枚）を牌画像で提示し、成立している役名を4択で問う | ◎ |
| FR-3.2 | 各問は**主役（最も大きい役）を答える**形式とする。役満や上級役は性質上どうしても下位の役を伴う（例: 大三元は必ず役牌を含む／字一色は四暗刻か七対子を伴う）ため、複合する場合は**最高位の役を正解**とし、誤答選択肢は**手牌に存在しない役**から選ぶ。完全な複数役の同時判定はフェーズ4で扱う | ◎ |
| FR-3.3 | 手牌はテンプレート＋ランダム化で毎回異なる見た目にする（丸暗記防止） | ◎ |
| FR-3.4 | **出題対象は「手牌の形から判定できる役」に限定**する。状況役（立直・一発・門前清自摸和・海底/河底）は本モードの対象外とする | ◎ |
| FR-3.5 | 提示する手牌は対象の役が**確実に成立**していること | ◎ |
| FR-3.6 | 解答すると即時に正誤・解説を表示し、セッション終了時に正答数を表示する | ◎ |

## 6. データ要件

### 6.1 牌の表記（内部表現）
| 種別 | 表記例 | 画像（FluffyStuff/Regular） |
|------|--------|------|
| 萬子 | `1m`〜`9m` | `Man1.svg`〜`Man9.svg` |
| 筒子 | `1p`〜`9p` | `Pin1.svg`〜`Pin9.svg` |
| 索子 | `1s`〜`9s` | `Sou1.svg`〜`Sou9.svg` |
| 東/南/西/北 | `E`/`S`/`W`/`N` | `Ton/Nan/Shaa/Pei.svg` |
| 白/發/中 | `haku`/`hatsu`/`chun` | `Haku/Hatsu/Chun.svg` |

### 6.2 役データ仕様（`lib/yaku.ts`）
各役は以下を持つ：`id` / 役名 / 読み / 飜数 / `tier`(1〜3) / 成立条件の説明文 / `recognizable`(手牌認識クイズ対象か) / 認識クイズ用テンプレート（`recognizable` の場合）。

### 6.3 対象25役（ティア別）
- **Tier1（初級・最頻出 / 10役）**: 立直 / 断么九（タンヤオ）/ 門前清自摸和（ツモ）/ 平和 / 役牌・白 / 役牌・發 / 役牌・中 / 自風牌 / 場風牌 / 一盃口
- **Tier2（中級 / 9役）**: 三色同順 / 一気通貫 / 混全帯么九（チャンタ）/ 七対子 / 対々和 / 三暗刻 / 混一色 / 一発 / 海底摸月・河底撈魚
- **Tier3（上級・役満含む / 6役）**: 清一色 / 小三元 / 大三元 / 国士無双 / 四暗刻 / 字一色

> 注: 以下の役は手牌の形だけでは確定できないため `recognizable=false`（暗記カード・4択クイズには出題するが、手牌認識クイズには出題しない。FR-3.4）。
> - **状況役**: 立直・門前清自摸和・一発・海底摸月/河底撈魚（和了のタイミングで成立）
> - **文脈依存**: 自風牌・場風牌（席・場が分からないと判定不能）
> - **門前形で表現不可**: 対々和（鳴きなしの4刻子は必ず四暗刻になり単独表現できない）
>
> 上記7役を除く18役が手牌認識クイズの対象。

### 6.4 ランダム化の制約
- **色シフト**（萬↔筒↔索）は `suitShiftable=true` の役のみ適用。混一色・清一色・字一色・国士無双など色が意味を持つ役は固定。
- **数字シフト**（順子の開始数字をずらす）は役ごとの制約に従う（例: 断么九は2〜8、チャンタ系は端を含む）。
- 許可された変換のみ適用し、変換後も役が成立することを保証する。

| ID | 分類 | 要件 |

| NFR-6 | 保守性 | 役・テンプレートはデータ駆動（`lib/yaku.ts`）で、役の追加・修正がデータ編集のみで可能 |

## 10. 受け入れ基準（フェーズ1 完了条件）

- [ ] ホームから3モードに遷移でき、各モードからホームに戻れる（FR-0.1）
- [ ] ティア選択で学習範囲が切り替わる（FR-0.2）
- [ ] 牌画像が全種類正しく表示される（FR-0.3）
- [ ] 暗記カードで選択ティアの役を閲覧し、表裏をめくれる（FR-1）
- [ ] 4択クイズが10問出題され、即時正誤・解説・最終スコアが表示される（FR-2）
- [ ] 手牌認識クイズで毎回異なる手牌が出題され、提示役が必ず成立している（FR-3.3, FR-3.5）
- [ ] 手牌認識クイズに状況役が出題されない（FR-3.4）
- [ ] `handGenerator` の単体テストが全て通る
- [ ] 375px 幅で表示が崩れない（NFR-2）
- [ ] Vercel にデプロイされ、公開URLをスマホで開いて操作できる（NFR-1, NFR-5）

## 付録A: フェーズ2〜4 概要（参考）

- **フェーズ2**: 進捗保存（覚えた役・成績）＋ 苦手復習モード（間違えた役を優先再出題）。スマホとPCで同期するため localStorage 不可 → API Routes + Prisma + DB + 簡易認証。
- **フェーズ3**: 段位別ロードマップ。雀魂の段位を単位に「必要なこと・優先度・今は不要なもの」を提示。静的版＋個人化版（現段位＋成績から自動判定）。
- **フェーズ4（候補・未定）**: 手牌認識クイズの完全ランダム化。既存ライブラリ（npm `riichi` 等）で役判定。複合役・「役なし」判定を扱う。自作エンジンは作らない。
